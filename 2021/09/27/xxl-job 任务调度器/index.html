xxl-job 任务调度器

官方地址 ： https://www.xuxueli.com/xxl-job/

github 源码地址 ： https://github.com/xuxueli/xxl-job

1.简介

​ 目前我们在项目中可能接触到定时任务框架quartz，应用也是比较广泛的，其也是支持分布式任务调度的，通过数据库竞争锁来实现，当然会有很多的局限性（可能这也是xxl-job出现的原因），quartz支持多种数据库，xxl-job其实也是在quartz的基础上实现的，但是修改了任务调度的模式，并且任务调度采用注册和RPC调用方式来实现。（在本质上来说，主要是要基于数据库 而xxl-job就只能够支持mysql数据库）

3.源码结构

​ 源码分为三个项目分别为：

​ xxl-job-core : 项目的核心，在自己的项目中也需要依赖次 maven 依赖。

​ xxl-job-admin: 此项目主要是做各种接口提供的。例如：添加任务，删除任务，修改。。。（如果需要源码修改，主要是修改这一部分）

​ xxl-job-executor-samples : 此项目是源码提供的一些简单的实例。

5.可视化界面介绍

我认为最主要的点在于 “任务管理” ，“执行器管理” 这两个是整个调度器可以正常调度的关键。

xxl-job特点：

​ 1.xxl-job最大的特点在于，它对于代码的耦合性极小，甚至可以说是没有。

​ 2.它们将 “调度规则” 和 “逻辑执行” 充分的独立。在admin中产生关联，但是在代码中没有任何相应的联系。

​ 3.再就是 它的 “调度规则” 极其丰富，基本可以达到大部分的业务要求。

​ 4.天生的分布式调度器。其中数据库中有一个 “锁表” 其主要是做分布式所用的。

任务管理器：

其中可以看出

​ 执行器 ： 是用来 规则 和 执行器相关联的配置。而这是一个大的概念，一个执行器可以有多个方法执行。所以就有个 JobHandler.

JobHandler : 这是在当前执行器中，配置需要调度的那个方法。在后续 java 集成xxl-job中会提到这个 JobHandler.

调度类型：这个有两个选项 “CRON” ,”固定速率” 。一般我们只会使用CRON，应为固定速率CRON也可以很好的实现。

Cron : 就是用来写Cron表达式。

路由规则：这个相对比较重要，对于 “高可用”，“削峰” 基本都是由他来控制，图中也很清楚。

阻塞处理策略：主要是避免在集群的情况下，任务重复调度。

执行器管理：

​ AppName : 表示执行器组名字。

​ 名称 ：组别名

​ 注册方式：

​ 自动注册：就表示当自己的服务启动后，服务配置了xxl-job配置，服务就会分配一个ip和一个端口自动注册到xxl-job中作为执行器。这多用在 ip 不固定，要反复扩容和缩容的项目。

​ 手动注册：就是提前手动将需要注册的ip都配置到调度器中，这多用于固定服务，已知服务的固定ip，并且ip不会轻易改变。

​ 机器地址： 就是各个执行器的ip地址。

​ 注意在 java 代码中的配置要和这里的组一致，才能够注册进去。也是通过这个来关联的。
