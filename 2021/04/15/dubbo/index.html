 Apache Dubbo是一个基于Java的高性能开源RPC框架 。

2.官方流程图

角色介绍

Registry : dubbo 是需要依托于注册中心来完成Rpc调用的。此角色为注册中心，其中默认是采用zookeep注册中心，其它的注册中心也是可以实现的。

Monitor : 这是对调用服务的监控。

Consumer : 表示服务的消费者。

Provider : 表示服务的提供者。

3.下载安装dubbo

1.下载zookeep服务

1.了解zoo.cfg配置

​ apache-zookeeper-3.5.7-bin 网上下载或官方下载都即可。

​ zookeep中核心配置 zoo.cfg 参数介绍

tick : 翻译成中文的话就是滴答滴答的意思，连起来就是滴答滴答的时间，寓意心跳间隔，单位是毫秒，系统默认是2000毫秒，也就是间隔两秒心跳一次。

tickTime的意义：客户端与服务器或者服务器与服务器之间维持心跳，也就是每个tickTime时间就会发送一次心跳。通过心跳不仅能够用来监听机器的工作状态，还可以通过心跳来控制Flower跟Leader的通信时间，默认情况下FL的会话时常是心跳间隔的两倍。

initLimit : 集群中的follower服务器(F)与leader服务器(L)之间初始连接时能容忍的最多心跳数（tickTime的数量）

syncLimit : 集群中flower服务器（F）跟leader（L）服务器之间的请求和答应最多能容忍的心跳数。

dataDir : 该属性对应的目录是用来存放myid信息跟一些版本，日志，跟服务器唯一的ID信息等。

2.下载服务监控

下载地址 ：https://github.com/apache/dubbo

1.找到 dubbo-admin-master.zip 并且解压 得到 dubbo-admin-master文件

2.进去到 dubbo-admin-master 下的 dubbo-admin项目此项目是spingboot编写的。

3.修改相关配置 ，找到 dubbo-admin 项目的 pom文件 并且将项目的 spingboot依赖修改成你当前项目使用的spingboot依赖。

4.找到 dubbo-admin 项目中的 application.properties 配置文件查询和修改相关配置

5.退回到有pom.xml文件的目录 使用maven来将项目打包 使用 mvn package 命令打包

6.将打包后的 jar 包使用 java -jar 命令运行

成功后 访问 http://localhost:7001/
