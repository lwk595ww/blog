ElasticSearch 学习文档

简述

ElasticSearch 是由Doug Cutting 创建的分布式搜索引擎，底层是封装的 Lucene 的搜索引擎。Lucene 是用 Java 编写的。

下载 ElasticSearch 服务

地址：

​ https://www.elastic.co/cn/downloads/?elektra=home&storm=hero

下载所需要工具

​ elasticsearch-7.11.2-windows-x86_64.zip （elasticsearch服务）

​ elasticsearch-head-master.zip （head elasticsearch集群和索引数据的可视化工具）

​ kibana-7.11.2-windows-x86_64.zip ( 工具的可视化工具 输入命令 )

​ elasticsearch-analysis-ik-7.11.2.zip （中文分词器）

启动 elasticsearch （windons）

1.解压后进去config 目录找到 elasticsearch.yml 文件 配置 （开启跨域支持，为head可视化工具连接做准备）

​ http.cors.enabled: true

​ http.cors.allow-origin: “*”

2.解压后进去 bin 目录 找到 elasticsearch.bat 文件 双击启动

启动 head-master

​ 1.解压后 使用 cnpm install 命令下载 所需要的依赖 （注意：它属于一个前端技术所支持 所以需要 node.js环境才可以运行）

​ 2.在 elasticsearch-head-master 目录下 使用 npm run start 启动

启动 kibana

​ 1.在 config 目录下找到 kibana.yml 并添加 i18n.locale: “zh-CN” 进行汉化

​ 2.在 bin 目录下 双击 kibana.bat 启动脚本

索引的结构

倒排索引：

在 elasticsearch 中自动维护了一个索引表 ，叫做 “倒排索引” 表示会在这个表中检索这个文档中有多少个词出现的次数，然后分别记录在这张表中，在查询的时候会在这张表中找到出现过的范围，在此范围中进行查询，就会大大提高效率。

ik分词器

什么是分词器：

​ 分词器： 就是将一组数据按照特定的规则分开检索。

​ 例如：“我爱中国” 分词器就会将这句话分为 “我” ，“爱” ，“中” ，“国” 这是由于默认的是外国了分词器，他们不支持中文分词。

​ ik 分词器就是适用于中中文的分词器。

​ 例如：“我爱中国”，就会分成 “我” ，“爱” ，“中国” 在此默认的字典中可以将 “中国” 定为一个词。

​ 分词也就相当于 mysql 数据库中的模糊查询。

ik分词器插件集成到 elasticsearch

   1.将ik分词器的压缩包放到 elasticsearch 中的 plugins 文件下 ，创建文件夹 ik 并且在ik文件夹下解压 ik 分词器的压缩包。（注意：一定要创建 名字为 ik 的文件夹 ，并且在此文件夹下解压文件。不然会启动失败，并且闪退）

   2.重启 elasticsearch 会看到 ik 插件被加载。
​

elasticsearch 结构

elasticsearch 结构和mysql中的结构相似，我们可以对比记忆

索引（indices） -> 库（databases）

类型 （types） - > 表 （tables）

文档 （document）-> 行 (rows)

fields -> 字段 （columns）

Rest 风格

​ 一种软件架构风格，而不是标准，只是提供了一组设计原理和约束条件。它主要用于客户端与服务端交互的软件。基于这个风格设计的软件可以更简洁，更有层次，更易于实现缓存等机制。
