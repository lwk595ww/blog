BigDecimal类型的数据

简述

​ BigDecimal 是 java 在 java.math 中提供的，主要是用来处理超过16为有效数字的数进行精度的运算。java中的基本类型数据像双精度double 是可以处理16位有效数据之内的数据进行精度计算的，但是在实际的开发中往往要计算的数据都会超过16位有效数字。而double在超过16位有效数字后就会有精度丢失问题，像涉及到金额方面重要的数据，精度丢失就会成为一个很严重的问题。而BigDecimal就在很好的程度上解决了这一问题。

BigDecimal 类型初始化总结

以上的问题可以看出，在初始化 double 类型的数据时 ，输出的结果跟我们想象的结果并不相同 。

原因：

​ 在对 double 类型进制转换问题，在存到内存里，小数会别转换成二进制，而二进制是不能准确标识 0.1 的 ( 这里是把0.1作为栗子 ) ，底层已经被转化成了超过8位的很长一长串二进制。

解决方案

​ 在开发的过程中，我们尽量将要初始化的数据转换成String类型的数据后，再实现初始化，这样就可以避免这一问题。

BigDecimal 两数比较

​ 描述：

​ 在一般我们用别的数据类型的时候，基本都是采用基本的比较符。例如：>(大于)，< (小于) , == (等于) ，!= (不等于) ，! (非) ，|| (或) ，&& (与) 等。

但是 BigDecimal 类型的数据就会发现并不能用这些基本的比较符来进行比较了，而他专门提供了一个方法来为此类型的数据做比较。

compareTo()
